
## План реализации расширенной админ-панели

### Этап 1: Новые таблицы в базе данных
- **fleet** — автопарк (название, статус free/busy/maintenance, пробег, заметки)
- **clients** — клиенты (ФИО, контакты, уровень лояльности, бонусы, статус black/white list)

### Этап 2: Расширение edge function `admin-bookings`
- Добавить действия: получение клиентов, автопарка, обновление статуса авто, управление бонусами

### Этап 3: UI админ-панели с табами
1. **📊 Аналитика** — выручка за день/неделю/месяц/год, кол-во бронирований, средний чек, загрузка парка (графики Recharts)
2. **📋 Бронирования** — текущая таблица (уже есть)
3. **👥 Клиенты** — таблица с фильтрами, история аренд, черный/белый список
4. **🚗 Автопарк** — статус каждого авто, пробег, переключение статуса
5. **👑 Лояльность** — уровни клиентов, бонусы, ручное начисление/списание
6. **🔔 Лента событий** — последние действия (новые брони, изменения)

### Технические решения
- Аналитика считается из данных бронирований (без отдельной таблицы)
- Клиенты создаются автоматически из бронирований + ручное управление
- Компоненты разбиты по файлам для читаемости
